# Spec: Task Domain Model

## Purpose
Єдине джерело істини для структури задачі та колекцій `Inbox`/`Today` у MVP.
Будь-яка зміна цієї моделі (нове поле, новий статус) спочатку правиться тут,
потім у `types.ts`.

## Requirements

### Task
Кожна задача має:
- `id: string` — унікальний ідентифікатор.
- `title: string` — текст задачі (результат парсингу AI або ручного вводу).
- `priority: 'low' | 'medium' | 'high'` — визначається AI при парсингу, за
  замовчуванням `'medium'`, якщо AI не зміг визначити.
- `dueDate: string | null` — ISO-дата дедлайну, якщо є.
- `scheduledTime: string | null` — час на сьогодні, якщо задача в `Today`.
- `status: 'inbox' | 'today' | 'done'` — поточний стан задачі.
- `source: 'text' | 'voice'` — звідки прийшов вхід.

### Inbox
- Колекція `Task[]` зі `status: 'inbox'`.
- Містить усе, що AI розпарсив, але користувач ще не розподілив на сьогодні.

### Today
- Колекція `Task[]` зі `status: 'today'`.
- Формується користувачем (або AI-пропозицією) із задач Inbox.
- Задача, позначена виконаною, переходить у `status: 'done'`, але не видаляється
  фізично (потрібно для майбутньої аналітики/історії).

## Scenarios

#### Scenario: AI не визначив пріоритет
- **WHEN** AI-парсинг не може впевнено визначити пріоритет тексту
- **THEN** задача створюється зі `status: 'inbox'` та `priority: 'medium'`

#### Scenario: Задача переноситься в Today
- **WHEN** користувач переносить задачу з Inbox у Today
- **THEN** `status` змінюється на `'today'`, `scheduledTime` може бути заповнений

#### Scenario: Задача виконана
- **WHEN** користувач позначає задачу виконаною
- **THEN** `status` змінюється на `'done'`, запис залишається в сховищі
